# Day 30 - Palindrome Linked List

## Problem
LeetCode 234 - Palindrome Linked List

## Approach
Used the Fast and Slow Pointer technique to find the middle of the linked list. Reversed the second half of the list in-place and compared it with the first half node by node. If all corresponding nodes matched, the linked list was identified as a palindrome.

## Time Complexity
O(n)

## Space Complexity
O(1)

## Learned
- Fast and Slow Pointer technique
- In-place linked list reversal
- Comparing two halves of a linked list
- Efficient palindrome detection
- Optimizing linked list problems with constant extra space
